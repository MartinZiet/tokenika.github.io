◆ transaction_create()

transaction_handle transaction_create ( )

create a pending transaction

This function allocates a native transaction and returns a handle to the caller. Within the context of a single transaction there is a soft limit to the number of pending transactions that can be open at a time. This limit is at the descretion of the Producers and exceding it will result in the current transaction context failing.

Returns
handle used to refer to this transaction in future API calls

◆ transaction_drop()

void transaction_drop ( transaction_handle  trx)

drop a pending transaction

This function will discard a pending transaction and its associated resources. Once a transaction is sent it can no longer be dropped.

This handle should be considered invalid after the call

Parameters
trx- the transaction_handle of the pending transaction to send
